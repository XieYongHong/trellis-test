# 项目页面设备健康子页

## Goal

在 `modules/page-ui` 中实现设备健康子页，让运维人员识别风险设备和健康评分下降原因。

## Requirements

* 页面挂载在 `page-ui` 的设备健康子路由。
* 展示项目整体健康评分。
* 展示风险设备列表，每条包含设备名、原因、评分和状态 chip。
* 健康评分只是状态判断，不做复杂分析钻取。
* 不渲染无业务用途的装饰图表。

## Acceptance Criteria

* [ ] 设备健康子路由可访问。
* [ ] 整体健康评分可见且有状态解释。
* [ ] 风险设备至少 3 条。
* [ ] 风险原因是面向运维人员的自然文案。
* [ ] 页面不包含编辑弹窗。

## File Scope

* `modules/page-ui/views/project/device-health/index.vue`

## Out of Scope

* 健康算法实现
* 设备巡检工单创建
* 多维归因钻取

## Definition of Done

* 只完成本任务范围内的实现或整理。
* 通过相关构建/类型检查，或明确记录未执行原因。
* 若发现范围需要扩大，先更新 PRD 或返回父任务拆解，不直接顺手实现。
